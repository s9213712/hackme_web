# 2026-05-14 Exp5 v17 90+ Run

## Summary

Exp5 reached `90.9949/100` under the v2 advanced score policy:

- runtime full credit up to `30000ms/game`, linear decay to `120000ms/game`;
- full gauntlet: `24W/6D/0L` across `30` complete games;
- threefold rate: `20.00%`;
- average elapsed: `14024.485ms/game`;
- grade: `advanced_engine_candidate`.

The default NNUE JSON weights were not changed. v17 is an engine-code/profile
upgrade, not a new main-model artifact.

## Model / Artifact Policy

Current conclusion: Exp5's main JSON behaves as static base evaluation
parameters. Score improvements have repeatedly come from search profile,
engine heuristics, trap priors, and conversion rules rather than checksum
changes.

Policy going forward:

- main-model JSON snapshots are only meaningful when
  `services/games/models/chess_experiment_5_nnue.json` checksum changes;
- v16 base snapshot is retained only because it was requested, and it is
  explicitly a duplicate static-base snapshot:
  `docs/games/model_snapshots/2026-05-14_exp5_nnue_v16_base_model_88464dba7e3497e7.json`;
- future generated artifacts should be adapter / experience table / small
  model candidates, audited separately from the static base model;
- production Exp5 now uses `fixed_depth_balanced` via
  `EXP5_PRODUCTION_SEARCH_PROFILE`.

## Implemented Changes

| Area | Change |
|---|---|
| Production profile | Route-level Exp5 dispatch now uses `fixed_depth_balanced` for deterministic strength. |
| Scoring | `chess_exp5_advanced_score.py` now supports configurable runtime policy; default is v2 `30s` full credit, `120s` zero credit. |
| Main model policy | Added `exp5_model_artifact_policy()` to document static-base vs generated adapter roles. |
| Pipeline defaults | Default chess pipeline no longer promotes/refines Exp5 main JSON unless explicitly requested. |

| Mate net | Added cheap forced-check detector for moves where the only reply permits mate-in-one. |

separate audited adapter / experience artifacts.
