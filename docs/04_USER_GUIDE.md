# 04 User Guide

一句話說明：這份文件給一般使用者，說明登入後主要功能怎麼用、哪些情況是正常限制、看到錯誤時該怎麼理解。

## 設計目的

部署者與管理員需要一份可以直接給使用者的精簡指南，而不是把 `WEB.md`
整份丟給對方。這份文件保留核心操作流程，讓一般人先能用，再需要時才進深層說明。

## 使用方法

### 使用者登入後通常會做的事

- 修改個人資料與密碼
- 依自己的喜好調整個人外觀，但不影響其他人
- 加入聊天室 / 社群 / 公告 / 討論區
- 使用 Cloud Drive、相簿、影片
- 視站點是否啟用，使用 ComfyUI、PointsChain、交易、遊戲
- 若收到治理通知或處分，可到 Appeals / Notices 檢視

### 最常見功能路線

#### 聊天 / 社群

- 建房、加房、發文、回覆、檢舉
- 主側欄已有「個人面板」：可查看自己的主頁、編輯公開資料、管理好友申請，或用好友代碼直接加入好友。好友代碼是系統產生的不可預測隨機代碼；只有自己看得到，可重新產生，舊代碼會失效。留言、貼文、影音、排行榜與遊戲紀錄的完整主頁入口仍在後續串接中。完整規格見 [USER_PROFILES_AND_FRIENDS.md](social/USER_PROFILES_AND_FRIENDS.md)
- 指定對象的功能會依身分過濾：一般使用者通常只能選已成為好友的人；root / manager 為站務目的可看到全站使用者，若對方也是自己的好友會置頂並特別標註
- 若你是 manager / root，已發布的公告現在也能直接回頭編輯，不必先刪掉重發
- 按 `建立聊天室` 後才會展開建立聊天室欄位；頁面不會常駐顯示整組表單
- 官方聊天群對一般使用者採匿名顯示，不顯示人數；root 顯示 `root`，管理員顯示編號管理員，兩者會以粗體並標示官方。root / manager 可為治理目的看到原發言者
- 一般群聊是否允許匿名由聊天室建立者決定；加入允許匿名的群聊時，可選擇自己是否匿名。匿名時名稱與頭像都會匿名化。一對一 PM 不提供匿名，因為對話本身已有明確對象
- 非好友不可直接 PM 或建立 private group；root / manager 為管理目的可私訊非好友用戶
- 聊天室表情包改成 emoji 快捷鍵，送出後會直接顯示實際 emoji，不再顯示「微笑 / 感謝」這類文字貼圖
- 聊天室附件現在直接掛在訊息輸入框旁：選檔後會立即加入待送清單，可送出前先刪除
- 聊天 / 私訊 / 公告附件在實際上傳時，會存到雲端硬碟 `/attachments/` 路徑；不是另外多一套附件系統
- 權限由角色與 member level 決定

#### Cloud Drive / 相簿 / 影片

- 先上傳檔案，再管理 visibility / albums / share link
- 雲端硬碟分成「檔案管理」與「容量管理」；容量使用狀態用水位式視覺顯示，不是電池圖示
- 在 Cloud Drive 檔案瀏覽器裡，現在可直接雙擊資料夾列進入，不必只靠右側 `開啟` 按鈕
- 大檔可使用分段上傳；若重新整理頁面，任務中心會保留未完成 session，但瀏覽器不會替你保留本機檔案權限，所以你需要重新選擇同一個檔案才能繼續上傳
- BT / direct link 下載會進任務中心，顯示下載速度、進度與可用度提示；可暫停、恢復或取消。任務中心採 lazy-load，只有進入該頁時才輪詢實際進度，並會優先顯示正在跑、失敗或需要操作的長任務。正在上傳、下載、HLS 處理、E2EE streaming 準備或 ComfyUI 生成時，登出或重新整理頁面不應中斷伺服器端任務
- 分享管理已移到「管理」下。分享列表可編輯分享選項、撤銷、查看存取紀錄；檔案分享連結除了下載，也可以在瀏覽器預覽。按下任何複製連結按鈕後，按鈕下方應顯示已完成複製
- 若分享對象要指定使用者，一般帳號只能選好友；root / manager 於站務功能可選全站使用者
- 相簿以連續照片流呈現，不在每張照片上常駐預覽 / 下載按鈕；滑鼠移到照片上會稍微放大，點選後進全頁檢視並可用左右按鈕切換
- 影片是建立在 Cloud Drive 上，不是第二套儲存系統
- 音樂檔現在可直接在雲端硬碟 inline 預覽；PDF 也會優先使用瀏覽器原生檢視器。若是 strict E2EE PDF，會先在瀏覽器端解密，再用內嵌檢視器或新分頁備援開啟。prepared HLS 影片在 Safari 走原生 HLS，桌機 Chrome / Firefox / Edge 會自動切到內建 `hls.js`；strict E2EE 影音若有建立 `E2EE Streaming v2` manifest，會以密文分段下載與瀏覽器端分段解密播放，否則退回舊版完整解密，所以首次載入與部分快轉情境仍可能較慢
- 若你在同一次登入 session 內連續開多個 E2EE 檔案，系統會先嘗試你剛剛成功過的最近密碼；只有真的解不開時才會再跳出詢問
- 壓縮檔預覽現在會顯示檔案/資料夾清單、大小、壓縮後大小與備註，不再只是一段文字明細
- 發布影音的欄位按下發布按鈕後才會展開。HLS 準備完成前，影音不會一直卡在公開列表的「準備中」；上傳者會看到處理提示，完成後收到通知。影音頁支援搜尋，做法接近常見影音平台的搜尋入口
- 若你發布的是 `持連結可看` 的 E2EE 影音，發布時只會在瀏覽器端要求你輸入一次原始 E2EE 密碼；之後可在影音頁的分享控制面板看到分享狀態、剩餘觀看次數、到期日、第二層分享密碼與重新產生 / 撤銷入口。若完整分享連結中的 fragment 遺失，伺服器無法幫你復原，只能重新產生分享；分享頁若啟用 `E2EE Streaming v2`，會明確顯示「讀取分享授權 / 下載加密影音 / 瀏覽器端解密」階段

#### ComfyUI

- 選模型、提示詞、LoRA、參數後產圖
- 需要時可改成 `img2img`、`inpaint`、`outpaint` 或 `upscale`
- `img2img / inpaint / outpaint / upscale` 可上傳來源圖；ControlNet 可另外上傳控制圖，並設定類型、強度與作用區間
- ControlNet 目前支援 `Canny`、`Depth`、`OpenPose`、`Lineart / Scribble`、`SoftEdge`、`Tile`
- 歷史清單可一鍵把先前的提示詞、LoRA、ControlNet、來源圖 / 遮罩圖設定套回再重跑
- `ComfyUI Workflow 工作台` 可把目前表單匯出成 workflow JSON，再匯入為個人 preset、日後一鍵套回/重跑；若你是 root，還可把自己的 preset 發布成官方 preset
- 產圖完成後可儲存到 Cloud Drive、分享到論壇或丟棄
- 頁面會明確顯示目前是本地模式還是遠端模式
- 產圖會建立背景任務；大模型載入慢、VRAM 不足或磁碟 I/O 慢時，狀態可能顯示後端暫時沒有回報進度，但主網站 request 不應同步等到當機
- workflow 匯入若含絕對路徑、shell/exec 節點、外部 URL 或缺少必要模型 / LoRA / ControlNet，系統會直接拒絕並顯示明確錯誤

#### 個人外觀

- 到 `修改資料 -> 個人外觀`
- 可以改字體風格、背景風格、面板風格、側邊欄寬度、顏色、密度、圓角、字級與內容寬度
- 這些設定只改你的畫面；root 設的全站預設仍是其他人的基準
- 若你想放棄自己的外觀覆寫，可直接按編輯視窗底部的 `恢復全站預設`，再按 `儲存`

#### PointsChain / Trading

- 站內點數、影片打賞、交易模組都走 PointsChain
- 交易是模擬與驗證用途，不是實盤交易所
- 交易市場現在由 root 在後台管理；因此不同站點可看到的市場可能不同，若某市場被停用，
  新下單入口會消失或被拒絕，但你既有的歷史與報表仍會保留
- 若 root 調整了原本預設的 seeded 市場，後台會標成 drifted；這不影響你看到的實際市場，
  因為執行期仍以站上的 market registry 為準
- 若畫面顯示「目前風控級價格不可用，已暫停市價單與高風險交易；限價單仍可使用」，
  代表展示 / 參考價還在，但融資、強平、市價單與 bot 風控用的 `risk-grade price`
  已降級；此時可改用限價單，或等風控級價格恢復
- 建立 Grid Bot 時，系統現在會先顯示每格毛利、手續費、扣費後淨利與損益兩平間距；
  如果顯示紅燈，代表扣完費預期虧損；黃燈則代表仍可能賺，但利潤太薄
- 交易手續費與借貸利息會先累積小數殘值；只有在現貨賣出、機器人停止、借貸結算或清算等真正結算時才轉成整數點數，且不足 1 點或有小數時無條件進位。借貸交易本身也有按名目金額計算的手續費，例如用 100 保證金借 400 買入，手續費以 500 名目金額計算
- Grid Bot 建立時應預留並凍結給 bot 操作的底倉、掛單用積分 / 現貨與預留手續費；停止 Grid Bot 時，系統應詢問要保留底倉或賣出
- Workflow bot 的可用金額以建立時設定的預算為基礎；若日後調整預算，減少額度不能低於已凍結、待成交、未結費用與風控保留所需金額
- 回測時若你先選開始或結束時間，系統會依目前週期直接提示另一側最遠可選到哪裡；
  超出單次範圍時，也會直接告訴你應該把哪一邊往前或往後縮

#### 遊戲

- 西洋棋若選一般內建難度，不會顯示 Stockfish depth 欄位
- 只有選 `Stockfish（本機）` 難度時才會出現 depth 欄位；後端會限制在 `1` 到 `20`
- 若這台主機沒有可用的 Stockfish binary，Stockfish 難度不會出現在可選清單

## 原理

- 許多使用者操作最後都會落到權限、PointsChain、Cloud Drive、或 feature flag 檢查。
- 因此「看得到按鈕」不等於一定有權限；真正可信的是後端重新驗證後的結果。
- 站點也會依 root 設定顯示或隱藏可選模組，所以不同部署的功能頁面可能不完全相同。

## 失敗情境與提示

- 顯示「此功能目前已由 root 關閉」：
  代表該模組未啟用，或你的角色不足。
- 個人外觀欄位是灰的，或儲存時被拒絕：
  代表 root 暫時關掉了個人外觀覆寫；你的既有個人外觀若先前有存，仍可能繼續套用。
- 想回到全站預設、但不想逐欄改回來：
  用視窗底部的 `恢復全站預設`，最後再按 `儲存`。
- 顯示權限不足：
  代表後端驗證沒有通過，不是單純前端顯示問題。
- 影片、檔案或預覽打不開：
  可能是 visibility、加密模式、權限、檔案狀態或舊 key 無法解密。
- ComfyUI 產圖中：
  產圖會在背景 job 執行，重新整理頁面不會讓伺服器改成同步等待；頁面會用 job 狀態顯示進度與完成結果。若 root 設的是遠端 API，模型下載工具本來就不會出現。
- 分段上傳重新整理後沒有繼續：
  請在任務中心選回同一個本機檔案。這是瀏覽器檔案權限限制，不代表伺服器任務遺失。
- 分享連結缺少 E2EE 片段金鑰：
  代表複製到的連結少了 `#...` fragment；伺服器看不到這段金鑰，只能重新複製完整連結或重新產生分享。
- 若頁面提示缺少 ControlNet 模型 / node、控制圖格式錯誤、或 `control strength` 超出範圍：
  代表後端已明確拒收這次設定，不是靜默失敗。

## 測試方式

- 以一般使用者身分依序測聊天、發文、上傳、影片、ComfyUI、通知、Appeals
- 驗證未登入、權限不足、功能關閉、資料不存在時的 UI 訊息
- 用 [11_QA_TESTING.md](11_QA_TESTING.md) 的手動逐步測試格式記錄頁面 / API / DB 對帳
