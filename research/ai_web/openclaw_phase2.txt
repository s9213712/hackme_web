任務：實作 OpenClaw Phase 2：Root-approved Controlled Automation Executor

目標：
讓 OpenClaw 不只產生分析報告，還能提出可執行的安全處置方案；但所有高風險操作必須 root 核准，且只能執行 allowlist 內的 server-side action。

核心原則：
- OpenClaw 不可直接執行 shell command。
- OpenClaw 不可直接存取 secrets。
- OpenClaw 不可直接修改 DB。
- OpenClaw 只能建立 action proposal。
- 實際執行由 hackme_web server-side executor 完成。
- executor 僅允許固定 allowlist actions。
- 高風險操作需 root approve / 二人覆核。
- 所有 proposal / approve / execute 必須寫 audit。

一、Action Allowlist

允許低風險：
- RUN_HEALTH_CHECK
- RUN_INTEGRITY_RESCAN
- RUN_RETENTION_DRY_RUN
- CREATE_DIAGNOSTIC_REPORT
- SUMMARIZE_SECURITY_EVENTS
- CREATE_MANUAL_SNAPSHOT

允許中風險，需要 root approve：
- ENABLE_MAINTENANCE_MODE
- DISABLE_DISCORD_SYNC
- DISABLE_MAIL_WORKER
- QUARANTINE_FILE
- REVOKE_USER_SESSIONS
- FORCE_PASSWORD_RESET
- TEMP_BLOCK_IP

高風險，需要二人覆核：
- RESTORE_SNAPSHOT
- ENTER_SUPERWEAK
- KEEP_SUPERWEAK_DIRTY_STATE
- APPROVE_HIGH_RISK_INTEGRITY_FINDING
- DELETE_USER
- MASS_ROLE_CHANGE
- ROTATE_MAINTENANCE_BYPASS_TOKEN

禁止：
- RUN_SHELL_COMMAND
- READ_SECRET
- MODIFY_ROOT_PASSWORD
- DELETE_AUDIT_LOG
- DISABLE_INTEGRITY_GUARD
- DOWNLOAD_PRIVATE_FILE
- DECRYPT_E2EE_FILE
- MODIFY_SMTP_SECRET
- MODIFY_DISCORD_TOKEN

三、Service

新增：

services/openclaw_executor.py

功能：
- validate_action_type(action_type)
- classify_action_risk(action_type)
- validate_payload(action_type, payload)
- run_preconditions(action_type, payload)
- dry_run_action(action_type, payload)
- execute_action(action_type, payload, actor)
- run_postconditions(action_type, payload)
- record_execution(...)

重點：
- payload schema 必須固定
- 不接受任意自由 JSON
- 不接受 shell command
- 不接受 raw SQL
- 不接受 arbitrary file path

四、Action Schema

每個 action 必須定義 schema。

例如：

TEMP_BLOCK_IP：
{
  "ip": "1.2.3.4",
  "duration_minutes": 30,
  "reason": "repeated login failure"
}

CREATE_MANUAL_SNAPSHOT：
{
  "snapshot_type": "manual",
  "notes": "before incident response"
}

ENABLE_MAINTENANCE_MODE：
{
  "reason": "security incident investigation",
  "eta": "unknown"
}

QUARANTINE_FILE：
{
  "file_id": "...",
  "reason": "suspected malware"
}

REVOKE_USER_SESSIONS：
{
  "user_id": 123,
  "reason": "suspected account compromise"
}

五、執行流程

OpenClaw：
1. 分析事件
2. 產生 action proposal
3. proposal 存 DB
4. root 在 UI 審核

Server：
1. root approve
2. executor 驗證 action allowlist
3. executor 驗證 payload schema
4. run preconditions
5. dry-run
6. execute
7. postconditions
8. audit
9. 寫 execution result

六、UI

OpenClaw Assistant -> Action Proposals

每筆 proposal 顯示：
- action type
- risk level
- rationale
- payload preview
- preconditions
- dry-run result
- required approval
- approve / reject / execute

高風險：
- 顯示紅色 danger zone

七、Release Check

scripts/release_check.py 新增：

- openclaw_enabled 若 true：
  - gateway URL 是否 loopback
  - redaction 是否 enabled
  - low risk auto action 是否 disabled
  - executor allowlist 是否載入
  - 禁止 action 是否不可執行
  - 是否有 pending high-risk proposal

八、測試

新增 tests/test_openclaw_executor.py

至少測：
1. 非 root 不可 approve/execute
2. 禁止 RUN_SHELL_COMMAND
3. arbitrary SQL 被拒絕
4. arbitrary file path 被拒絕
5. low-risk action dry-run 成功
6. high-risk action 未 approve 不可執行
7. two_person action 未達票數不可執行
8. payload schema 錯誤會拒絕
9. execute 失敗會記錄
10. audit log 必有

九、驗收

完成後必須證明：
- OpenClaw 可提出行動方案
- root 可 approve/reject
- executor 只執行 allowlist
- 高風險需要二人覆核
- 所有執行都有 audit
- OpenClaw 無法執行 shell / SQL / secret read
