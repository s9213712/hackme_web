任務：研究並設計 hackme_web 與 OpenClaw 整合模組

目標：
將 OpenClaw 整合為 root-only 輔助系統，用於：
1. 審計報告分析
2. 安全事件摘要
3. Integrity Guard finding 解讀
4. snapshot / restore 建議
5. 伺服器健康狀態分析
6. 危急狀況處置建議
7. 可選的受控自動化操作

核心原則：
- 僅 super_admin/root 可使用。
- OpenClaw 不得直接繞過 RBAC。
- OpenClaw 不得直接讀取 raw secrets。
- OpenClaw 不得自動執行高風險操作。
- 所有 AI 分析與建議需寫入 audit。
- 所有 tool action 必須有明確 allowlist。
- 危險操作必須 root 二次確認或二人覆核。

三、system_settings

新增：

openclaw_enabled = false
openclaw_gateway_url = "ws://127.0.0.1:18789"
openclaw_require_root_approval = true
openclaw_allow_low_risk_auto_actions = false
openclaw_max_context_events = 500

安全：
- gateway URL 只允許 loopback 預設。
- 若設成 LAN，release_check 必須警告。
- 不允許從 UI 顯示 OpenClaw token / pairing secret。

services/openclaw_redaction.py

六、AI 分析功能

新增 root-only API：

POST /api/root/openclaw/analyze/audit
POST /api/root/openclaw/analyze/security-events
POST /api/root/openclaw/analyze/integrity
POST /api/root/openclaw/analyze/health
POST /api/root/openclaw/analyze/release-readiness
GET /api/root/openclaw/reports
GET /api/root/openclaw/reports/<id>

每個 analyze：
1. 收集 bounded context
2. redaction
3. 組成 prompt
4. 呼叫 OpenClaw
5. 儲存 report
6. 寫 audit

七、受控操作提案

OpenClaw 可以提出，但不可直接執行：

- enable maintenance mode
- create snapshot
- run health check
- run integrity rescan
- recommend reject/approve integrity finding
- recommend retention dry-run
- recommend disabling Discord/Mail sync
- recommend IP block review

高風險操作必須進 openclaw_action_proposals：
- restore snapshot
- enter superweak
- keep dirty superweak state
- approve high-risk integrity finding
- delete user
- mass settings change
- clear audit/security events

八、禁止 OpenClaw 自動執行

禁止：
- 讀取 raw secrets
- 修改 root 密碼
- 下載使用者私有檔案
- 解密 E2EE
- 刪除 audit log
- 清除 integrity finding
- restore snapshot
- enter superweak
- 修改 RBAC / roles
- 修改 SMTP / Discord token
- 執行 shell command
- push code 到 GitHub

九、低風險可選自動化

若 openclaw_allow_low_risk_auto_actions=true，才可允許：

- create diagnostic report
- run health check
- run integrity rescan
- run retention dry-run
- create snapshot
- summarize security events

仍需：
- audit log
- tool event log
- rate limit
- root session active

十四、驗收標準

完成後必須證明：

1. root 可以看到 OpenClaw gateway 狀態。
2. root 可產生 audit/security/integrity/health 分析報告。
3. 報告不含 raw secrets。
4. OpenClaw 不可被 manager/user 使用。
5. OpenClaw 只能提出高風險操作，不會直接執行。
6. 所有分析與提案都有 audit。
7. OpenClaw gateway 掛掉時網站正常運作。
8. release_check 能檢查 OpenClaw 設定風險。

十五、最重要限制

OpenClaw 是 root 的「分析與建議助手」，不是自動 root shell。

若未來要加入自動執行，也必須走：
OpenClaw proposal -> root approve -> server-side allowlisted executor -> audit log
