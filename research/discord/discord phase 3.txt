任務：實作 Discord 管理通知與查詢指令（只讀 + 通知，不給權限操作）

目標：
- 將關鍵事件推送到 Discord 管理頻道
- 提供 Discord slash commands 查詢站內資訊
- 嚴格禁止 Discord 直接執行管理動作（只讀）

一、DB Migration

新增：
discord_command_logs
- id INTEGER PRIMARY KEY
- discord_user_id TEXT
- command TEXT
- args_json TEXT
- status TEXT (success/failed/denied)
- result_preview TEXT
- created_at TEXT

system_settings 新增：
- discord_admin_channel_id
- discord_command_enabled = false
- discord_command_rate_limit_per_user_per_min = 5

二、Service

新增 services/discord_commands.py

功能：
1. handle_command(payload)
2. verify_user_binding(discord_user_id)
3. check_rate_limit(discord_user_id)
4. execute_command()

支援指令：

/status
→ 回傳：
- server mode
- health readiness
- pending high-risk integrity findings
- snapshot status

/latest
→ 最新公告或討論串

/report
→ 回報問題（寫入本站 reports）

/me
→ 顯示綁定狀態 + member level

三、Discord Interactions

新增 route：
POST /api/integrations/discord/interactions

- 驗證 Discord signature
- 回應 PING
- 處理 application commands

回應格式：
- ephemeral（只給發送者）
- 禁止 public sensitive info

四、管理通知（Outbound）

在以下事件觸發 Discord 通知：

CRITICAL：
- Integrity Guard high risk finding
- snapshot restore failed
- audit chain failure
- superweak entered

WARNING：
- repeated login failure
- upload high risk
- moderation queue backlog

通知格式：

[CRITICAL] Integrity Guard Finding
Time: ...
Detail: ...
Link: <admin URL>

五、API

新增：
GET /api/admin/discord/commands/logs
GET /api/admin/discord/alerts

六、安全

- 指令必須綁定 Discord user
- manager/root 才能看到管理資訊
- 所有回應過濾敏感資料
- 不允許任何 write 操作

七、測試

- 未綁定 user 無法使用 command
- rate limit 生效
- CRITICAL alert 能發送
- webhook/token 不出現在 log

八、驗收

- /status 可用
- 管理頻道可收到 alert
- 非管理者無法查詢敏感資訊
