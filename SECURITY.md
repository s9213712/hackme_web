# Security Policy

This repository keeps a root `SECURITY.md` so GitHub can detect the project
security policy on the default branch.

Current security controls, supported reporting flow, and known limitations are
documented in:

- [docs/SECURITY.md](docs/SECURITY.md)
- [docs/security/PENTEST.md](docs/security/PENTEST.md)
- [docs/security/FUNCTIONAL_PERMISSION_PENTEST.md](docs/security/FUNCTIONAL_PERMISSION_PENTEST.md)

Please do not open public issues for unpatched exploitable findings. Use the
project owner's private reporting channel first, then publish details only after
the fix is available.
