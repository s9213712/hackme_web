    價格來源與融合比例
    
                  
    設定交易執行價來源，並查看目前真實融合占比與排除來源。
    
                
    
                  
              
    
                
    
                  
    
                    
                    
                    
    預設會用多交易所融合價格；若部分 API 失效，會自動用仍健康的交易所重新分配權重，必要時再退回最後健康快取。
    
                  
    
                  
    
                    
                    
                    
    自動模式會依各交易所掛單簿深度決定占比；手動模式可由 root 個別調整各交易所權重。
    
                    
    深度分數會統計各交易所前 N 檔，並計算中間價附近 ±1% 範圍內的買賣盤名目量；系統會取較弱一側作為 depth score，再自動正規化成 100% 的融合比例。
    
                    
    這套融合價格目前較適合作為 v1 即時參考價與 sanity check，不建議單獨作為強平、機器人或實際成交的唯一依據。
    
                  
    
                  
    
                    
                    
    
                      
                      %
                    
    
                    
    auto_depth 會以 midpoint 為中心，盡量覆蓋這個價格區間。若來源無法覆蓋完整區間，診斷區會標示 truncated。
    
                  
    
                  
    
                    
                    
                    
    可設定 top 10 / 50 / 100 / 200 / 500 / 1000。不同交易所可實際提供的最大 depth 不同；若碰到供應商上限，診斷區會標示 provider depth limit reached。
    
                  
    
                  
    
                    
                    
    
                      
                      %
                    
    
                    
    若買盤或賣盤覆蓋不到這個深度，來源會被排除；覆蓋超過門檻但未達完整區間時，會標示 truncated。
    
                  
    
                  
    
                    
                    
    
                      
                      %
                    
    
                    
    避免單一交易所占比過高；若可用來源太少而無法滿足上限，診斷區會明確標示。
    
                  
    
                  
    
                    
                    
                    
    低於這個數量時，融合價格會改標示為保守模式，不建議用於高風險判斷。
    
                  
    
                  
    
                    
                    
                    
    只要至少有這麼多家交易所 API 正常回傳，就可依 root 下方開關決定要不要自動暫停交易。預設 2 家。
